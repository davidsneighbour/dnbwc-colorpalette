# dnbwc-colorpalette

`@davidsneighbour/dnbwc-colorpalette` provides the `dnb-colorpalette` web component. The initial prototype reads colors from the component content and renders a responsive, equally spaced color grid that can be dropped into an Astro site or any other page that can load an ESM web component.

The component intentionally does not bundle Tailwind CSS. It uses Tailwind utility class names in the generated markup and assumes the implementing site has Tailwind enabled and loaded.

## Features

* Defines the `dnb-colorpalette` custom element.
* Reads palette colors from the inner text content of the component.
* Supports one color per line with a configurable parser format attribute.
* Accepts any CSS color value supported by the current browser, including hex values, named colors, `rgb()`, `hsl()`, and newer CSS color functions when the browser supports them.
* Trims empty lines.
* Ignores lines that begin with `//`, `/*`, or `*` after trimming.
* Renders one grid column for every parsed color.
* Uses the color value as both the column background and visible label.
* Gives every color column equal width with CSS grid.
* Adds `is-component` and `dnb-wc-colorpalette` to the generated outer `div` for custom styling hooks.
* Provides configuration attributes for the outer class, maximum width, column minimum width, height, border classes, and an optional copy button.

## Installation

Install the package from npm once it is published:

```bash
npm install @davidsneighbour/dnbwc-colorpalette
```

For local development from a checkout of this repository:

```bash
npm install
npm run build
```

## Basic usage

Import the component once in your JavaScript entry point:

```js
import '@davidsneighbour/dnbwc-colorpalette';
```

Then add one CSS color per line inside the component:

```html
<dnb-colorpalette>
  #0f172a
  hsl(221 83% 53%)
  rebeccapurple
  tomato
</dnb-colorpalette>
```

The rendered palette contains one equally sized column per color. Each column uses the input line as its CSS `background-color` value and as the visible label.

## Astro installation and usage

Install the package in your Astro project:

```bash
npm install @davidsneighbour/dnbwc-colorpalette
```

Create or update a client-side entry file, for example `src/scripts/colorpalette.js`:

```js
import '@davidsneighbour/dnbwc-colorpalette';
```

Load that script in your Astro page or layout:

```astro
---
import '../scripts/colorpalette.js';
---

<dnb-colorpalette outer-class="mx-auto" max-width="48rem">
  #111827
  oklch(70% 0.16 250)
  hsl(142 71% 45%)
  gold
</dnb-colorpalette>
```

If your Astro project uses a different bundling pattern, import the package in any script that is included on pages where the custom element appears. Tailwind classes generated by the component are useful only when Tailwind CSS is already configured for the Astro site.

## Content format

The default format is `lines`:

```html
<dnb-colorpalette format="lines">
  /* comments are skipped */
  #f8fafc
  // comments are skipped
  rgb(15 23 42)
  blue
</dnb-colorpalette>
```

Rules for `format="lines"`:

* Split inner text content by newline.
* Trim each line.
* Remove blank lines.
* Remove lines that start with `//`, `/*`, or `*`.
* Keep lines that are valid CSS color values for the current browser.

Additional formats are planned, but not implemented yet.

## Attributes

| Attribute | Default | Description |
| --- | --- | --- |
| `format` | `lines` | Parser format for the inner content. Only `lines` returns colors in the prototype. |
| `outer-class` | empty | Extra classes appended to the generated outer `div`. |
| `max-width` | empty | Optional inline `max-width` for the generated outer `div`, such as `48rem` or `100%`. |
| `min-column-width` | `0` | Minimum width used for each grid column and each column style. |
| `min-height` | `50px` | Minimum height of the palette grid. |
| `max-height` | empty | Optional maximum height of the palette grid. |
| `border-class` | `border border-slate-200` | Tailwind classes applied to the grid border. |
| `copy` | absent | Boolean attribute that renders a copy button when present. |
| `copy-label` | `Copy palette` | Visible copy button label. |

## Styling

The generated structure is light DOM, not shadow DOM, so the host page can style the palette and Tailwind utility classes can apply normally.

The generated outer wrapper always contains these classes:

```txt
is-component dnb-wc-colorpalette
```

Example with custom classes and sizing:

```html
<dnb-colorpalette
  outer-class="mx-auto my-8"
  max-width="64rem"
  min-column-width="6rem"
  min-height="120px"
  max-height="16rem"
  border-class="border-2 border-slate-900"
>
  #020617
  #2563eb
  #22c55e
  #facc15
</dnb-colorpalette>
```

## Copy button

Add the `copy` attribute to render a button that copies the parsed palette to the clipboard as one color per line:

```html
<dnb-colorpalette copy copy-label="Copy this palette">
  red
  white
  blue
</dnb-colorpalette>
```

Clipboard access depends on browser support, secure context requirements, and user permissions.

## Development

Install dependencies:

```bash
npm install
```

Run tests:

```bash
npm test
```

Build the distributable module:

```bash
npm run build
```

Serve the standalone dist test page over HTTP after building:

```bash
npm run serve:test
```

The test page must be served over HTTP because browsers block ES module imports from local `file://` pages.

Run all configured checks:

```bash
npm run lint
```
